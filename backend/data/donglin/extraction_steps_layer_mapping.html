§4 同一条规则在 A / B 两套方法里的完整溯源

挑 AR-RULE-001「应收账款贷方余额重分类」这条规则，对比两套方法分别怎么处理：

方法 A 这条规则怎么进本体

  1. 步骤 1：人读 CSA 1311 应收账款审计准则 + 甲所 2024-01 内部沉淀 → 写到 implementation_guide.md
  2. 步骤 5：人写 07_audit_procedures.md 把"客户级期末贷方余额需重分类"明确为甲所做法
  3. 步骤 6：build_instances.py line 485-489 编码： 
    "AR-RULE-001", "应收账款贷方余额重分类", "应收应付",
"IF customer.closing_balance < 0 THEN 重分类至 2203",
"high", "事务所", "甲所", "2024-01",
"客户级期末贷方余额 = 客户预付/超付 → 移到「预收款项」",
"closing_cr > 0"  # formal_expression
  4. 步骤 7：agent_fill.py 跑 A6 时, FOR 每个客户 IF closing_cr>0 THEN 触发 → 153 客户命中, 累计 ¥7.11M 建议重分类

层级标签：L2 法理 CSA 1311 → L3 形式化 甲所阈值 → L0 实例化 甲公司 153 命中

方法 B 这条规则怎么进本体

  1. 步骤 2a: extract_excel.py 扫到 A6-2!I8 的公式 =D8+G8-H8 并把这条写到 cell_formulas.jsonl#L14627
  2. 步骤 3a：Team 3 设计 schema_v3.md 时, 看 A6-2 第 I 列 = 期初+借-贷 即"调整后" → 决定建 :AuditAdjustingEntry Class
  3. 步骤 4b：build_fill_rules.py 生成: 
    :fillrule-a62-i8-from-d8g8h8 a :FillRule ;
    :hasRuleKind "formula" ;
    :hasFormulaExpression "=D8+G8-H8" ;
    :hasEvidenceRef "cell_formulas.jsonl#L14627" ;
    :outputField :field-a62-i8 ;
    :inputField :field-a62-d8, :field-a62-g8, :field-a62-h8 .
  4. 缺失：没有规则说"如果 D8+G8-H8 < 0 触发重分类" → 这是业务语义，需要叠 A 才能补

层级标签：L3 公式 A6-2!I8 的求值规则 (机械的, 不知道是"重分类")
未覆盖：L1+L2 的 CSA 1311 / 重分类语义

同一条业务规则的两个视角
  • A 知道 "这是 CSA 1311 应收账款审计准则要求的，甲所 2024-01 沉淀了 closing_cr>0 形式化阈值"
  • B 知道 "A6-2!I8 这格 = D8+G8-H8 (Excel 公式) — 这是底稿模板里 hardcoded 的算法"
  • 完整 = A + B: A 提供 为什么这样算，B 提供 具体在哪格算的
  • 如果只有 B，给客户看 A6-2!I8 的公式他知道是怎么算的，但不知道"为什么算" → 无法 review
  • 如果只有 A，知道"重分类是规则"但不知道甲所模板把这步放在了 A6-2 的 I 列 → 写回 .xlsm 时找不到位置

§6 启示 — 这套分析回答了 3 个问题

  1. "哪些层是机器抽出来的，哪些是人脑里来的" — L3 甲所专有 这层在 A 里全是人脑/字面量, 在 B 里全是机器抽。 L1+L2 在 A 里是人 + LLM 设计, 在 B 里是人从证据反推。 L0 客户事实 在 A 里有 (openpyxl 读 TB/Aux/Voucher), 在 B 里几乎没读 (B 只读模板不读数据)。
  2. "为什么需要 A + B 才能闭环" — A 缺L3抽取自动化, B 缺L1+L2准则映射 + L0客户事实。 A 知道"为什么这样填", B 知道"具体填到哪格"。 A 算数, B 写到 .xlsm 模板里。
  3. "L4 怎么补" — 两种方法都缺 L4, 但有不同进入路径:
    • A 的入口: ChatPanel 对话日志 + 用户改写 Agent 填的内容
    • B 的入口: VBA 源码里"审计师定制的宏" (369 条 VBA assignment 部分含审计师手写偏好)
    • 合流: 双管道 LearningInbox — 对话事件 + VBA 模式 → ProposedRule → 累积 ≥ 3 次 → 审计师 sign-off → 升级 L3
给项目经理展示这张表的话术

"侯老师，这是我对两套抽取方法的步骤拆解。 我们用的方法 A 强在 L1+L2 准则建模 + L0 客户数据 (Agent 能算能填)， v3 用的方法 B 强在 L3 自动抽甲所模板 (可重跑 + 证据链)。 把两个串起来 = 真正能跑生产的本体: A 算数, B 把数字写回你熟悉的 .xlsm 模板， 再加 L4 经验沉淀, 就是你跟我说的『审计本体长期资产化』的路径。"
