§4 业务场景实例 — A6 应收账款审计

用一个具体场景演示 TBox 和 ABox 如何在审计中配合：

场景 A6 应收账款审计 — Agent 自动填稿过程

Agent 接到任务"填 A6 应收账款"。它从 TBox 知道"这种底稿要做什么"，从 ABox 知道"江苏大王这家公司有哪些应收账款数据"。

📐 TBox 提供"做什么" — 类型骨架

  • WorkingPaper 类型 — 定义"什么叫底稿，要有 sheet_data / audit_conclusion / review_status"
  • Account 类型 — 定义"什么叫科目"
  • SubLedger 类型 — 定义"什么叫辅助账"
  • AuditRule 类型 — 定义"什么叫规则，要有 formal_expression"
  • Sampling 类型 — 定义"什么叫样本"
  • RelatedPartyRisk 类型 — 定义"什么叫关联方资金占用"

上面 TBox 全是 L1+L2 / L3 类型，跨所通用

🔗 LinkType 定义关系

  • PaperConcernsAccount — "底稿 → 涉及科目"
  • SubLedgerForAccount — "辅助账 → 科目"
  • PaperAppliesRule — "底稿 → 应用规则"
  • RuleTriggersAdjustment — "规则 → 触发调整"

⚡ ActionType 定义动作

  • Read — 读取 TB / Aux / Voucher
  • Reconcile — 比对账面 vs 辅助账
  • Sample — 抽样 Top 5 客户
  • Generate — 生成调整分录

📋 ABox 提供"是什么" — 具体实例

  • WorkingPaper 实例：L0 A6 货币资金 (id=94, 江苏大王专属)
  • Account 实例：L0 1122 应收账款 (江苏大王 TB 期末 ¥3,360,975.70)
  • SubLedger 实例：L0 808 客户辅助账行
  • AuditRule 实例：L3 AR-RULE-001 (东林内规, formal: closing_cr>0)
  • RelatedPartyRisk 实例：L0 黄燕红 ¥811,739.84

🔗 LinkInstance 具体连接

  • PaperConcernsAccount: A6 → Account-1122
  • SubLedgerForAccount: 808 行 → Account-1122
  • PaperAppliesRule: A6 → AR-RULE-001 / RP-RULE-001
  • RuleTriggersAdjustment: AR-RULE-001 → Z6-AI-A6-01 (¥7.11M)

📑 Agent 跑出的产物（也是 ABox）

  • A6.sheet_data.summary（审定表填好）
  • A6.sheet_data.customer_detail（458 客户行）
  • A6.sheet_data.aging_analysis（305 客户 FIFO 账龄）
  • 3,978 条 cell 追溯（含 153 个 RuleDerived）
  • 1 笔 Z6-AI-A6-01 调整分录 ¥7,114,390.40
关键洞察：Agent 跑 A6 时同时引用了 TBox 和 ABox ——
  • TBox 告诉它"应收账款审计要 Read + Reconcile + Sample + apply AR-RULE-001"
  • ABox 告诉它"江苏大王的 1122 期末是 ¥3.36M，要从 808 辅助账行抽数据，153 个客户命中 AR-RULE-001"
  • 填稿时每个数字都引用：📦 OT::SubLedger(TBox 类型) + 客户 000860(ABox 实例) + 📜 AR-RULE-001(ABox 规则实例)
