§1 诚实开场白（5 分钟）

⚠ 一定要诚实开场，不要让侯总误以为本体全部来自甲所

开场话术（请按这段背诵）

"侯老师，今天给您看的是一套本体平台 + AI Agent 自动填稿原型，用甲公司 2025 年度审计作为 demo 案例。

需要先说清楚 — 它分 3 类内容：

① 真实跑出来的产物：5 张已填底稿 / 4,250 个单元格的本体追溯 / 9 条异常识别 / 2 笔调整分录建议 — 这些是 Agent 用真实客户数据 (TB / 辅助账 / 12,620 笔凭证) 跑出来的。

② 公开准则相关：CSA 1141 / 1221 / 1311 / 1411 等 5 条审计规则 — 规则文字大致符合中注协/财政部公开准则，但形式化阈值是我们设的，需要您确认阈值符合本所实务。

③ AI 从模板形态反推：7 条标"甲所"的内规 (如 AR-RULE-001 应收贷方重分类、GC-INDICATOR-001 关联担保识别) — 这些是我们看到您的 442-sheet 底稿里有'重分类'/'担保识别'这些列后反推的，并不是从甲所内规手册抽出来的。这些请您重点 review，告诉我们是否符合本所实操、阈值要不要改。

今天 review 的目标：让您看清这套本体长什么样，提出 ①缺哪些规则 ②哪些规则阈值要改 ③哪些对象类型本所没有要删掉。这一轮 review 后，规则库才算真正属于甲所。"

不要说的话

  • ❌ "这是我们从您给的材料里抽出的 14 条审计规则" — 错，14 条里 7 条是 AI 推测的
  • ❌ "我们已经把甲会计师事务所的本体抽出来了" — 错，TBox 是设计的，不是抽取的
  • ❌ "这是您所审计师确认过的规则库" — 错，本次 review 才是首轮审计师确认

§3 Review Checklist · 6 个决策点

把这 6 个 checklist 打印给侯总，让他边看 demo 边在纸上标 ✓ 同意 / ✗ 不同意 / 🔧 需修改。

① AI 推测 7 条 L3 "甲所内规" 是否符合本所实务？

这 7 条规则的 source="甲所" 是我们贴的标签，规则内容是从模板形态反推的，需要您逐条确认。

AR-RULE-001 应收账款贷方余额重分类 · closing_cr > 0 · 命中 153 客户 ¥7.11M
问：阈值是 0 还是有最低金额？重分类目标科目是 2203 预收款项还是其他？
AP-RULE-001 应付账款借方余额重分类 · closing_dr > 0 · 重分类至 1123 预付款项
问：本所 B6 应付审定时是否就是这套做法？
RP-RULE-001 关联方资金占用披露 · is_related_party AND |balance| > PM · 触发 ZK5 单独披露
问：阈值是 PM 还是 PM/2？关联方判定标准是 CAS 36 还是有内规细化？
INV-RULE-001 存货归零异常 · 年度借方+贷方 > 500 万 AND 期末归零 → 反舞弊关注
问：500 万这个阈值合理吗？本所是否有类似的反舞弊规则？
TAX-RECLASS-001 出口退税反向余额 · code=1133 AND closing < 0 AND |closing| > TE
问：本所遇到 1133 反向余额是否就是重分类到 22210106？
GC-INDICATOR-001 关联担保覆盖率 100% → 持续经营关注
问：本所触发持续经营关注的标准是什么？还有别的指标吗（如流动比 / 速动比）？
DL-RULE-001 三级复核签字完整性 · 编制人 + 经理 + 合伙人三签
问：甲所是 3 级还是 4 级（含一级复核-洪潇）？

② 公开准则 5 条 L2 规则的阈值是否符合本所采用版本？

这 5 条规则文字符合公开准则，但具体阈值/形式化表达式是我们设的。

CASH-RULE-001 银行账面 vs 询证函 · 差 < ¥0.01 视为一致
问：本所容忍度是 ¥0.01 还是更宽松（如四舍五入到元）？
CASH-RULE-002 大额现金审批 · 单笔 ≥ ¥500,000 须有审批文件
问：本所大额阈值是 50 万还是其他？
REV-RULE-001 收入截止 · ±5 工作日窗口
问：是 5 工作日还是 5 自然日？或者 ±10？
FA-RULE-001 折旧重算 · CSA 1221 四级梯度 (SUM/TE/PM/Material)
问：本所 PM 计算基准是总资产 1% 还是有其他偏好？
TAX-RULE-001 业务招待费 5‰/60% 孰低
问：直接引用企税法第 43 条 · 您所是否有补充规则（如审计程序）？

§4 Demo 收尾 + 收集反馈（10 分钟）

收集反馈的方式

  1. 纸版 checklist：让侯总拿着我们打印的 §3 6 个 review item，边看边在每条旁写 ✓ / ✗ / 🔧
  2. 关键问题清单（让他口头回答 5 个问题）：
    1. "7 条 L3 内规里，哪几条阈值要改？改成什么？"
    2. "还缺哪些规则本所常用但本体没建？"
    3. "2 笔自动调整分录合规吗？借贷方向 / 科目代码 / 金额对吗？"
    4. "9 条异常少了什么？还有什么本所会重点关注的异常类型？"
    5. "10 大簇 ObjectType 分类合理吗？还有什么大类我们漏了？"
  3. 录音/录像（先征得同意）：方便后续整理反馈
  4. 3 周后跟进会：把侯总的反馈落实到本体后，再给他看一次升级版

把反馈落到代码

反馈处理流程:
  1. 反馈点 → 写到 backend/data/donglin/review_feedback_v1.md
  2. 规则阈值调整 → 改 build_instances.py 的 rules = [...] · 重跑 seed
  3. OT 增删 → 改 object_types.json · 重启 backend
  4. 缺失异常 → 加到 seed_anomalies.py 的 DONGLIN_ANOMALIES
  5. 修改完 commit + tag v1.1-hou-reviewed 标记侯总已 review
