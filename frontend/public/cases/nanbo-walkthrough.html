2穿行测试怎么做：方法、检查项与判定标准 ★ 审计师复核重点

本节说明本次穿行测试的测试设计与判断规则。请审计师评判：测试方向、检查项、阈值与结论判定规则是否符合本次审计目标。

2.1 测试认定与方向

测试认定：销售收入的发生/存在——验证已签销售合同确有真实的实物出库与之对应。
测试方向：「从合同到明细账」正向追踪——选定合同 → 追踪其出库 →（本次范围止于出库；发票/记账凭证/明细账不在范围，相应环节留空注明）。

2.2 穿行步骤（每笔样本）

  • 第一步：选取样本合同，核对合同基本要素（合同号、客户、货物、规格、数量、金额、签约日期）。
  • 第二步：追踪该合同客户对应的出库台账（出入库流水账）。
  • 第三步：逐项勾稽出库与合同的一致性（见 2.3 检查项）。
  • 第四步：评价勾稽结果，按 2.4 规则定结论；不一致或无法直接勾稽则保守地交审计师判断。

2.4 结论判定规则

  • 出现 「高」 级异常（如跨客户串单）→ 发现异常/风险
  • 出现 「中」 级异常 / 缺关键证据 / 综合置信 < 0.5 / 跨客户 / 弱关联 → 需人工复核
  • 无上述情形 → 通过测试
★ 请审计师评判：上述检查项是否齐备、阈值（数量 5% / 金额 1%）是否合适、结论判定规则是否稳妥。 如本次审计目标需更严或更宽（如增加"出库须可定位到本合同PO批次"为硬性检查、或调整容差），可据此修订规则后重跑。

3材料关联（link）怎么判断：配对规则 + 校验 + 逐组判断 ★ 审计师复核重点

「把哪份合同与哪份出库视为同一笔业务」是穿行测试最关键的判断——它直接决定后续勾稽与回填是否正确。本节把配对与校验的规则显式列出，并给出本次 4 组 link 的逐组判断，供审计师评判依据是否充分。

3.1 怎么"配对"（建立关联）

  • 主依据 · 源文件名数字前缀：送审材料以 1/2/3/4 编号一一对应（1 号合同 ↔ 1-1 号出库）。这是送审时人为建立的对应关系，作为最强配对信号。
  • 辅助核对 · 归一化客户名：去掉括号与"有限公司/股份/集团/厂"等通用后缀后，比较合同客户与出库客户的"客户核"是否互相包含（如"盐城正泰"⊂"盐城正泰新能源科技有限公司"）。
  • 绝不做：不仅凭合同号或客户名相同就自动归并；不跨客户合并。

3.2 风险标记规则

  • 跨客户(cross_customer)：同组出现互不包含的两个客户核 → 判跨客户，拆分不合并（最严风险）。
  • 弱关联(weak_link)：仅有前缀、客户名无法核对一致 → 标弱关联、提示人工。
  • 缺单：缺合同或缺出库一方。

3.3 人工确认（不自行拍板）

关联草稿必须经审计人员逐组确认（认可 / 修改 / 拆分 / 剔除）后，才用于后续 link 与回填。本次 4 组均无风险标记，人工已逐组确认采纳，并特别确认「盐城正泰新能源」与「正泰新能科技」名称相近但确为两家独立客户、未串单。

3.4 配对成立后怎么"校验"该 link 可靠

配对成立 ≠ 自动通过。系统用第 2 节的检查项校验该 link 是否一致可靠：客户须同一主体、数量/金额可比、计量口径可折算、日期顺序合理。不一致或无法直接勾稽 → 判"需复核"，把最终判断交审计师。
